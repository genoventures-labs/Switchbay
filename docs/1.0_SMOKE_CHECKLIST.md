# Switchbay 1.5 Smoke Checklist

Use this as the final hand-run pass before calling a build 1.5-ready. The goal is not exhaustive QA; it is proving the main rig works after install.

## Setup

- [x] Upgrade/install from Homebrew:

```bash
brew upgrade switchbay
switchbay version
```

Expected: prints the intended version.

- [x] Confirm basic help works:

```bash
switchbay --help
```

Expected: command exits cleanly and lists engines, skills, memory, knowledge, trace, radar, handoff, and MCP helpers.

- [x] Confirm no repo dirt before testing:

```bash
git status --short
```

Expected: clean or only intentional local test files.

## Workspaces

- [x] Check active workspace:

```text
/workspace
/workspace list
```

Expected: active cwd, repo root, branch, dirty files, and known workspace list render clearly.

- [x] Check workspace hop alias against a known repo:

```bash
switchbay "Say which workspace you are in." --hop Switchbay
```

Expected: one-shot mode starts in the matched workspace or prints a clear no-match/whitelist message.

## Workspace Knowledge

- [x] Build the local knowledge map:

```bash
switchbay knowledge refresh
switchbay knowledge
```

Expected: `.switchbay/knowledge/index.json` exists; file/chunk counts are non-zero.

- [x] Search the knowledge map:

```bash
switchbay knowledge search "approval gates"
```

Expected: returns sourced snippets with `path:start-end` line spans.

- [x] Check TUI commands:

```text
/index
/search approval gates
```

Expected: `/index` shows index status; `/search` returns sourced snippets.

## Trace Ledger

- [x] Before a turn, confirm trace status:

```bash
switchbay trace
```

Expected: shows latest trace or says no trace exists yet.

- [x] Run a one-shot turn after knowledge refresh:

```bash
switchbay "Using Workspace Knowledge if useful, tell me where approval gates are documented."
```

Expected: response completes and writes a trace.

- [x] Inspect the trace:

```bash
switchbay trace
switchbay trace export
```

Expected: trace summary shows runtime, workspace, knowledge source count, tool count, token estimates, and a JSON path under `.switchbay/traces/`.

- [x] Open the JSON trace path and spot-check:

Expected fields:

- `userPrompt`
- `runtime`
- `workspace`
- `context.knowledgeSources`
- `actions.tools`
- `actions.pendingApprovals`
- `result.finalAnswer`

## Operator Radar And Handoff

- [ ] Run Radar:

```bash
switchbay radar
```

Expected: prints blockers, warnings, clean signals, and a next command when friction exists.

- [ ] Run Handoff:

```bash
switchbay handoff
```

Expected: prints workspace, latest objective/session, changed files, trace/test signal, Daily Board, and next step.

- [ ] Check TUI/conversational equivalents:

```text
/radar
/handoff
Switchbay, run radar
Switchbay, write a handoff for next time
```

Expected: slash and conversational forms return local-first summaries without a model call.

## Tool Bridge And Approvals

- [x] Run a read-only local request:

```bash
switchbay "List the top-level files in this repo and summarize what kind of project this is."
```

Expected: Switchbay can inspect files and answer without asking for approval.

- [x] Trigger a safe approval gate in TUI:

```text
Switchbay, run git push so I can verify the approval gate, but do not actually push unless I approve.
```

Expected: Switchbay stages approval. `1`/`n` rejects, `2`/`y` approves, `3` marks yes-always for that command.

- [x] Reject the action:

```text
1
```

Expected: approval clears and no push runs.

## Final Release Readiness

- [x] Run source checks:

```bash
bunx tsc --noEmit
bun test
bun run build
```

Expected: typecheck passes, tests pass, build succeeds.

- [x] Confirm clean git state:

```bash
git status --short
```

Expected: clean except ignored `.switchbay/` runtime data.

- [x] Confirm install/update instructions:

```bash
brew upgrade switchbay
switchbay version
```

Expected: Homebrew installs the intended release version.

## Pass Criteria

1. Cloud lane answers.
2. Local lane either answers or gives a clear LM Studio setup error.
3. Workspace status/list/hop surfaces work.
4. Workspace Knowledge refresh/search works.
5. Trace Ledger writes and summarizes receipts.
6. Read-only tools run without approval.
7. Risky shell action stages approval and can be rejected.
8. Engine/Skills/Memory/Guide commands render.
9. Source tests/build pass.
