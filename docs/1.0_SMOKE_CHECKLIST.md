# Switchbay 1.0 Smoke Checklist

Use this as the final hand-run pass before calling a build 1.0-ready. The goal is not exhaustive QA; it is proving the main rig works after install.

## Setup

- [ ] Upgrade/install from Homebrew:

```bash
brew upgrade switchbay
switchbay version
```

Expected: prints the intended version.

- [ ] Confirm basic help works:

```bash
switchbay --help
```

Expected: command exits cleanly and lists engines, skills, memory, knowledge, trace, and MCP helpers.

- [ ] Confirm no repo dirt before testing:

```bash
git status --short
```

Expected: clean or only intentional local test files.

## Cloud Lane

- [ ] Confirm keys are available:

```bash
echo "$OPENAI_API_KEY" | wc -c
echo "$ANTHROPIC_API_KEY" | wc -c
```

Expected: non-zero output for at least one provider key.

- [ ] Run a tiny one-shot cloud request:

```bash
switchbay --lane cloud "Reply with one short sentence saying the cloud lane works."
```

Expected: a short assistant response, no empty turn.

- [ ] Open the TUI:

```bash
switchbay
```

Inside TUI:

```text
/model
/lane
```

Expected: model drawer opens; lane toggles Cloud/LM Studio cleanly.

## Local LM Studio Lane

- [ ] Confirm LM Studio env points to the real host:

```bash
echo "$SWITCHBAY_LMSTUDIO_BASE"
echo "$SWITCHBAY_LMSTUDIO_MODEL"
```

Expected: base URL uses the machine/IP LM Studio exposes, not a stale placeholder.

- [ ] Confirm local model list:

Inside TUI:

```text
/model local
```

Expected: drawer shows loaded LM Studio models or a clear setup/auth message.

- [ ] Run a tiny local request:

```bash
switchbay --lane local "Reply with one short sentence saying the local lane works."
```

Expected: local model responds or gives a clear LM Studio connection/auth error.

## Workspace Knowledge

- [ ] Build the local knowledge map:

```bash
switchbay knowledge refresh
switchbay knowledge
```

Expected: `.switchbay/knowledge/index.json` exists; file/chunk counts are non-zero.

- [ ] Search the knowledge map:

```bash
switchbay knowledge search "approval gates"
```

Expected: returns sourced snippets with `path:start-end` line spans.

- [ ] Check TUI commands:

```text
/index
/search approval gates
```

Expected: `/index` shows index status; `/search` returns sourced snippets.

## Trace Ledger

- [ ] Before a turn, confirm trace status:

```bash
switchbay trace
```

Expected: shows latest trace or says no trace exists yet.

- [ ] Run a one-shot turn after knowledge refresh:

```bash
switchbay "Using Workspace Knowledge if useful, tell me where approval gates are documented."
```

Expected: response completes and writes a trace.

- [ ] Inspect the trace:

```bash
switchbay trace
switchbay trace export
```

Expected: trace summary shows runtime, workspace, knowledge source count, tool count, token estimates, and a JSON path under `.switchbay/traces/`.

- [ ] Open the JSON trace path and spot-check:

Expected fields:

- `userPrompt`
- `runtime`
- `workspace`
- `context.knowledgeSources`
- `actions.tools`
- `actions.pendingApprovals`
- `result.finalAnswer`

## Tool Bridge And Approvals

- [ ] Run a read-only local request:

```bash
switchbay "List the top-level files in this repo and summarize what kind of project this is."
```

Expected: Bay can inspect files and answer without asking for approval.

- [ ] Trigger a safe approval gate in TUI:

```text
Bay, run git push so I can verify the approval gate, but do not actually push unless I approve.
```

Expected: Switchbay stages approval. `1`/`n` rejects, `2`/`y` approves, `3` marks yes-always for that command.

- [ ] Reject the action:

```text
1
```

Expected: approval clears and no push runs.

## Engine Bay, Skills, And Plugins

- [ ] Check Engine Bay:

```bash
switchbay engines
switchbay engines list
```

Expected: shows cache status and registered/cached engine files, or clear sync instructions.

- [ ] Check Skills:

```bash
switchbay skills
switchbay skills list
```

Expected: built-in or synced skills list, including review/debug/test-style skills.

- [ ] Check Plugins:

```bash
switchbay plugins
switchbay plugins list
```

Expected: shows the workspace plugin path and either installed plugins or a clear empty-state message.

- [ ] Check TUI drawers:

```text
/engines
/skills
/plugins
```

Expected: engine and skill drawers open; plugins prints status or installed manifests.

## Memory, Rules, And Quick Starts

- [ ] Check memory:

```bash
switchbay memory
switchbay memory refresh
switchbay memory facts
```

Expected: memory status works; refresh creates/updates `.switchbay/memory/`.

- [ ] Check guides:

```text
/quickstarts
/rules
```

Expected: built-in quick starts/rules render, including tool use and Switchbay MCP guidance.

## MCP Bridge

- [ ] Show MCP config:

```bash
switchbay mcp
```

Expected: shows user config path under `~/.switchbay/lmstudio.mcp.json`.

- [ ] Toggle bridge in TUI:

```text
/mcp on
/mcp off
```

Expected: runtime badge/messages reflect bridge enabled/disabled.

- [ ] Confirm catalog guardrail:

```text
/mcp catalog
```

Expected: shows trusted MCP catalog and does not invent unknown server ids.

## Final Release Readiness

- [ ] Run source checks:

```bash
bunx tsc --noEmit
bun test
bun run build
```

Expected: typecheck passes, tests pass, build succeeds.

- [ ] Confirm clean git state:

```bash
git status --short
```

Expected: clean except ignored `.switchbay/` runtime data.

- [ ] Confirm install/update instructions:

```bash
brew upgrade switchbay
switchbay version
```

Expected: Homebrew installs the intended release version.

## Pass Criteria

1. Cloud lane answers.
2. Local lane either answers or gives a clear LM Studio setup error.
3. Workspace Knowledge refresh/search works.
4. Trace Ledger writes and summarizes receipts.
5. Read-only tools run without approval.
6. Risky shell action stages approval and can be rejected.
7. Engine/Skills/Memory/Guide commands render.
8. Source tests/build pass.
